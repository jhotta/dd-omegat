Libraries

There are many libraries available to help you interact with the Datadog API.

C#

  • dogstatsd-csharp-client - A C# DogStatsD client.

Go

  • datadog-go - A Go DogStatsD client.

JAVA

  • java-dogstatsd-client - a DogStatsD Client for Java written by Indeed and Datadog.

Python

  • datadogpy - A Python Datadog API wrapper and DogStatsD client.

Ruby

  • DogApi - A Ruby Datadog API wrapper.
  • dogstatsd-ruby - A Ruby DogStatsD client.

PHP

  • php-datadogstatsd - An extremely simple PHP DogStatsD client written by Alex Corley.

Community Libraries

Some great folks have written their own libraries to help interact with Datadog. Check them out:

C#

  • metrics.net-datadog - a .NET translation of the metrics-to-datadog java adapter

Crystal

  • statsd.cr - A statsd client library implemented in Crystal by Mike Fiedler

Delphi

  • datadog-delphi - A statsd client library implemented in Delphi.

Elixir

  • ExStatsD - an Elixir DogStatsD library by CargoSense.
  • dogstatsd-elixir - a dogstatsd client in Elixir by Adam Kittelson.
  • mtx - an Elixir Datadog client by synrc.

Go

  • ecsdog - a standalone Go application that scrapes metrics and events from ECS, and sends them to statsd by ejholmes
  • godspeed - a feature-rich dogstatsd client written in Go by PagerDuty.
  • go-datadog-api - a Go wrapper for our API by Mark Smith from Dropbox.
  • go-dogstatsd - a dogstatsd client written in Go by Ooyala.
  • xstats - a generic client for service instrumentation using dogstatsd in Go by Olivier Poitrey.

Java

  • metrics-datadog - a backend to yammers’s metrics library written by Coursera.
  • Lassie - a Java screenboard API client by Bazaarvoice.
  • java-dogstatsd-client - DogStatsD Client for Java to submit both Events and Metrics written by arnabk.

Node.js

  • hotshots - a Node.js client for statsd, DogStatsD, and Telegraf written by Brightcove.
  • node-datadog - a Node.js API client, contributed by HashGo.
  • node-dogstatsd - a Node.js DogStatsD client, contributed by Young Han Lee.
  • node-dogapi - a Node.js API client, contributed by Brett Langdon.
  • datadog-metrics - Node.js API client, contributed by Daniel Bader.

Perl

  • webservice-datadog - a Perl API client, contributed by Jennifer Pinkham.
  • dogstatsd-perl - a Perl DogStatsD client, contributed by Stefan Goethals.

PHP

  • dog-statsd - A fork of thephpleague/statsd with additional Datadog features by Graze.
  • plesk_metrics_datadog - a PHP script to collect metrics from Plesk by Israel Viana.

Python

  • scales_datadog - a Datadog backend for the Scales library, written by Tommaso Barbugli.

Ruby

  • metricks-dogstatsd - a backend for the popular Metriks gem, written by Mavenlink.
  • hotdog - A command-line interface contributed by Yuu Yamashita.

R

  • datadogr - a simple R package to query for metrics.
  • rdog - an R package to analyze Datadog metrics into R.

Scala

  • datadog-scala - a Scala API client, written by Cory Watson.

Community Tracing (APM) Libraries

C#

  • DatadogSharp - A C# Datadog client that supports DogStatsD and APM.

Elixir

  • spandex - A Datadog APM reporting library.

Java

  • apm-client - A Java client for Datadog APM.

OpenTracing

  • datadog-tracer-js - OpenTracing tracer implementation for Datadog in JavaScript.

  • dd-go-opentracing - OpenTracing tracer implementation for Datadog in Go.

PHP

  • dd-trace-php - A PHP tracing implementation for Datadog.

Zipkin

  • dd-zipkin-proxy - A simple Zipkin-to-Datadog proxy.

Community Integration Libraries

Ansible

  • In addition to our official integration, the monitoring section of the ansible-modules-extras repository contains modules that interact with Datadog.

Consul

  • Publish consul service counts into Datadog via dogstatsd with this library.

Dogscaler

  • Scale up auto-scale groups based on the results of a datadog query with Dogscaler.

FreeSwitch

  • This is for a FreeSwitch ESL application to export statistics to DataDog using the dogstatsd API and is written by WiMacTel.

Google Analytics

  • You can get data into Datadog from Google Analytics using our API with this library.

Logstash Output

  • Logstash Output for Datadog
  • Logstash Output for Dogstatsd

NGINX LUA

  • Emit custom metrics directly from NGINX configurations using the nginx_lua_datadog module in your LUA scripts.
  • lua-resty-dogstatsd is an extension developed by mediba inc, which enables emiting metrics, events, and service checks to DogStatsD protocol. lua-resty-dogstatsd is released as GPLv3 and relies on the nginx cosocket API.

Phusion Passenger

  • Send health metrics from Phusion’s Passenger server using the passenger-datadog-monitor written by Stevenson Jean-Pierre

Pid-stats

  • This library will allow you to generate process information from StatsD, given pid files. It was created by GitterHQ.

Saltstack

  • Datadog Saltstack Formula
  • Datadog Saltstack written by Luca Cipriani.

If you’ve written a Datadog library, write us at code@datadoghq.com and we’ll be happy to add it to the list.
