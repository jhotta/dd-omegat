Agentの基本的な使用方法 (Ubuntu)

概要

このガイドでは、Datadog Agentの基本的な機能を説明します。 まだDatadog Agent のインストールを済ませていない場合は、Datadog にloginした状態で、該当OSの Agent ページに移動し、インストールの手順を確認してください。

Datadog Agent の起動と停止

手動で Datadog Agent を起動する:

sudo /etc/init.d/datadog-agent start

Datadog Agentを停止する:

sudo /etc/init.d/datadog-agent stop

Datadog Agentを再起動して、設定ファイルを再度読み込ませる:

sudo /etc/init.d/datadog-agent restart

動作ステータスの確認

Datadog Agent が起動しているかを確認する: (since 3.8.0)

sudo /etc/init.d/datadog-agent status

Datadog Agent のステータスに関する情報を収集する:

sudo /etc/init.d/datadog-agent info

エラーをトレースするために-vフラグをinfoコマンドに追記し実行する: (ver. 3.8.0 ~)

sudo /etc/init.d/datadog-agent info -v

設定ファイルの保存されているディレクトリ

Datadog Agent の設定ファイルの保存先 /etc/dd-agent/datadog.conf

インストールしたIntegrations の設定ファイルの保存先 /etc/dd-agent/conf.d/

トラブルシューティング

infoコマンドを実行することによって、Datadog Agentのステータス情報を表示することができます。

Datadog Agentに内包されている機能の個別ログは、以下のファイルに記録されています:

  • /var/log/datadog/supervisord.log (since 3.8.0)
  • /var/log/datadog/collector.log
  • /var/log/datadog/dogstatsd.log
  • /var/log/datadog/forwarder.log


上記の方法を実行しても問題が解決しない場合は、Datadogのサポートチームが問題解決のお手伝いをすることができます。Datadogのサポートチームに連絡を取るには、次のような方法があります。

ZenDeskの知識共有サイト

ZenDeskの知識共有サイトから、Datadogの使い方をしらべることができます。

Datadogのイベントストリーム上からのサポート

Datadogのダッシュボード上にあるイベントストリームから、@support-datadogで、サポートチームに連絡を取ることができます。この方法は、グラフのスナップショットを撮って質問したり、特定のイベントに関して質問する際に便利な方法です。更に米国東海岸時間の通常営業時間帯には、ライブチャット機能をDatadogの全てのページに設けています。

Emailによるサポート

support@datadoghq.comへEmailを送信することで、Datadogのサポートチームと連絡を取ることができます。

IRCによるサポート

freenodeの#datadog部屋で、Datadongのサポートチームや他のDatadogユーザに聞くこともできます。(このリンクから、 web チャットクライアントを利用することもできます。)

ステータスページとtwitterによるサービス状況の確認

Datadogのステータスページとオペーレーションチームのtwitterアカウントでは、パフォーマンス障害の最新情報を公開しています。
